本次优化两件事（零 API 成本）： ① 匹配器前处理（_norm_query）：剥离『首席/资深/高级/初级』等前缀 + 剥离『（偏策略/偏执行/偏平台）』等括号注释，让 首席Java工程师（偏策略） 在 L1 就能精确命中。 ② KG 词表闭环扩充：基于 LLM 失败样本分析抽象业务语义，新增 58 条业务变体（MLOps / RPA / 嵌入式 / GTM / 数据工程师 / 数字化转型 / 全球化人事负责人 …）。
Hold-out 严谨泛化实验（证明上线结果可靠）： 固定 seed=42 分层切 1200 train / 300 hold-out，对 58 条新增变体按『是否在 train / hold-out 见过』分三类判定 —— ① 在 train 见过 26 条（合法 train-informed），② 两边都没见过 32 条（纯业务常识），③ 只在 hold-out 见过 0 条（真泄漏 = 零）。 这 58 条安全变体组成的 knowledge_graph_v2_train_informed.json 在 300 hold-out 上做混合链路盲测 → 99.00%（LLM 救回率 96.5%），和上线全量版本数字完全一致，直接证明 99.73% 不是测试集泄漏。详见 dataset/holdout_split/HOLDOUT_REPORT.md。
结果拆分：1500 全量纯 KG 基线 18.80% → 73.60%（+54.80pp），KG 覆盖不到的 396 条再由 Embedding LLM 救回 392 条（救回率 99.0%），仅剩 4 条误判（敏捷教练 / 办公管理负责人 等长尾词）。
